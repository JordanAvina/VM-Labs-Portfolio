1. Objective
---------------------------
- Install Windows 11 Pro on a VirtualBox VM.

- Log in as domain user.

3. Steps Completed
---------------------------
01. Opened VirtualBox, created Windows 11 Pro VM (Windows 11 Lab 01).
02. Mounted Windows 11 ISO, started installation.
03. Selected Windows 11 Pro, accepted license terms.
04. Turned off internet connection to bypass Microsoft account requirement (initial attempt failed).
05. Opened CMD and ran OOBE\BYPASSNRO to bypass online account requirement.
06. Created local user account Jordan during setup.
07. Renamed Windows 11 VM to Desktop01.
08. Created second Windows 11 Pro VM (Windows 11 Lab 2).
09. Renamed second Windows 11 VM to Desktop02.
10. Started Server 2022 VM.
11. Configured Server 2022 and Windows 11 Lab 01 to use Host-only Adapter in VirtualBox network settings.
12. Set Static IP on Server 2022:
    - IP: 192.168.56.10
    - Subnet: 255.255.255.0
    - Default Gateway: 192.168.56.1
    - DNS: 192.168.56.10
    - Alternate DNS: 192.168.56.1
13. Set static IP on Windows 11 Lab 01 (Desktop01):
    - IP: 192.168.56.20
    - Subnet: 255.255.255.0
    - Default Gateway: 192.168.56.1
    - DNS: 192.168.56.10
    - Alternate DNS: 192.168.56.1
14. Verified network connectivity from Windows 11 Lab 01:
    - Ran ping javinit.com
15. Verified Server 2022 IP configuration using ipconfig.
16. Joined Desktop01 to domain javinit.com:
    - Right-click This PC > Properties > Change > Domain > javinit.com.
    - Entered domain admin credentials.
    - Restarted VM to apply changes.
17. Verified Desktop01 appeared in Active Directory Users and Computers on Server 2022.
18. Logged into Desktop01 using domain user Damian (password change at first logon).
19. Enabled Remote Desktop on Server 2022:
    - Right-click This PC > Properties > Rename this PC (advanced) > Remote tab
    - Check Allow remote connections to this computer.
20. Opened Remote Desktop Connection (mstsc) on Windows 11 Lab 01.
21. Started a remote session to Server 2022 using IP 192.168.56.10.
22. Successfully connected, then ended the remote session.
